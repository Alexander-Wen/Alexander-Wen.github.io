Fluid

Making a website is hard

I am lazy

The reason I made fluid was because I was lazy. When I was first making my personal website with different pages, I soon realized that if I ever made a change to the core of my website, I would have to copy and paste my changes into every html file that made up my website. Like any programmer, I thought to myself, "why don't I just make a program do this for me".

Designing Fluid

When designing fluid, I realized that every one of my html files was more of less the same. So I took all the features that would be the same for every html file, such as navbar and footer, and I put that into a template file. Everything that was variable was put into a body file, and I made my own syntax to be able to parse it properly. Below is an example of the body code.

ECE 150

In ECE 150, progrmamming, we were taught file reading and writing, and linked lists in C#. To practice in this new language, I decided to make this entire program in C# using file I/O, and each section is stored within a linked list. Although this might not be the most efficient way of doing it, I decided to do it this way, just because I wanted to practice for ECE 150.
